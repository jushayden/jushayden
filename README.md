## What I'm building

**[claude-code-memory-cache](https://github.com/jushayden/claude-code-memory-cache)** — persistent, token-efficient memory for Claude Code. Five cooperating layers (vector store, fact files, Obsidian vault, code graphs, per-repo brain files) kept fresh by hooks, plus a live 3D graph of the memory that pulses when sessions think. Measured on my own installation: architecture questions cost 22× fewer tokens through the code graph than reading source, and the docs publish the failures alongside the wins — [docs/STATS.md](https://github.com/jushayden/claude-code-memory-cache/blob/main/docs/STATS.md).

The rest of my homelab runs on two Raspberry Pis over Tailscale: a self-hosted voice assistant with local Postgres + pgvector, a task board, and kiosk displays for the graph.
